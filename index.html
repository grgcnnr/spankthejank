A more real life example

Layout Thrashing

With irresponsible JavaScript

Layout Thrashing occurs when 'something' reads from then writes to the DOM multiple times causing document reflows each time.

Paint Storms

Pretty much the worst thing ever

Paint storms sound cool but they are dumb

They occur when something causes a big chunk of the document to be repainted continuously

Caveats

  • Every browser is (a bit) different
    I've been talking about Blink in Chrome on Windows 7
  • Repaints aren't the end of the world
    (especially if the browser isn't busy)
  • GPU compositing isn't free
    We have limited bandwidth and VRAM
  • "Native" implimentations aren't always fastest
    (but they should get faster)
  • * {transform: translateZ(1)}
    Ha ha, no. Let browsers make up their own minds
  • Browsers are CSS is getting smarter
    Look out for the   will-change   property in the dev channel of all major browsers
  • Tools, not rules
    Every browser is different and they change all the time. Test.
